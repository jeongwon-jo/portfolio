(3) Tailwind.css

v 4.0

  • Tailwind.css 사용 이유 & 구현 방식

  • Maintenance-friendly: Tailwind.css는 클래스 유틸리티 기반이라 스타일 변경이 빠르고, 구조가 단순해 협업과 유지보수가 수월합니다.
    특히 프로젝트 인수인계 과정에서 CSS 관련 설명이 거의 필요 없었을 만큼, Tailwind의 높은 유지보수성을 직접 경험했습니다.
  • Consistent UI System: 색상·여백·그리드 등 디자인 토큰을 코드로 통일하여 안정적인 디자인 시스템을 구축할 수 있습니다.
  • 경량화된 빌드: 실제 사용하는 클래스만 남기므로 CSS 번들 크기가 작아 퍼포먼스도 우수합니다.
  • Tailwind는 처음 접했을 때 완전하게 생소했지만, 기존 bootstrap 경험을 기반으로 인프런 강의를 통해 독학(기본 CSS뿐 아닌 반응형 & 인터랙션까지)하며 탄탄하게 다졌습니다.
    이후 메인 페이지, 헤더/푸터/사이드바, 위젯, 테마 컬러링까지 모든 퍼블리싱 요소를 Tailwind로 구현하며 UI 전체를 유연하게 관리할 수 있는 구조로 구축하였습니다.

(4) Components

UI 요소를 컴포넌트 단위로 분리하는 것이 생산성과 유지보수성을 높이는 핵심입니다. 같은 Input이나 Table 구조를 여러 페이지에서 쓸 때도, 하나의 컴포넌트만 관리하면 전체 UI의 통일성이 자동으로 유지되고, 변경 사항이 빠르게 반영됩니다.

저는 퍼블리싱 단계에서 이러한 컴포넌트 기반 구조를 선제적으로 구축하여, Input, Table, List, Button 등을 모두 재사용 가능한 형태로 제작했습니다.
덕분에 개발자들은 디자인을 맞추기 위해 HTML/CSS를 새로 작성할 필요 없이 미리 만들어둔 컴포넌트를 그대로 불러와 개발에 집중할 수 있었고, 실제로 개발 단계에서 UI 관련 작업 시간이 크게 단축되었습니다.

Input 컴포넌트

상태별 스타일 처리(disabled / readOnly): disabled, readOnly 상태일 때의 스타일을 분리하여 사용 불가 상태에서도 UI의 일관성을 유지하도록 구성.

입력값 전체 삭제 기능(onClear): Input 내부에 Delete(초기화) 버튼을 기본 제공하여 사용자가 입력한 값을 한 번에 지울 수 있음.

재사용성과 유지보수성 향상: Input의 주요 속성(onChange, onFocus, onBlur 등)을 Prop으로 전달받도록 제작하여 어떠한 페이지나 폼에서도 동일한 컴포넌트를 그대로 사용할 수 있음.

Button 컴포넌트

일관된 디자인 시스템 유지: 프로젝트 전반에서 버튼 스타일을 통일하기 위해 variant(색상/스타일), size(크기) 를 분리해 하나의 컴포넌트에서 관리할 수 있도록 구성.

테마(theme) 연동된 동적 스타일 처리: ThemeProvider와 연동되어, 특정 테마(예: yellow theme)일 때는 primary 버튼의 텍스트 색상을 자동으로 조정하는 등 동적 스타일 적용 기능을 포함.

공통된 disabled 상태 처리: 버튼 컴포넌트에 disabled:opacity-50, cursor-not-allowed 등 공통 스타일을 내장하여 모든 버튼의 비활성화 UX를 일관되게 적용할 수 있도록 구현. 각 화면에서 따로 CSS를 작성할 필요가 없어 개발 효율성을 높임.

Search Period 컴포넌트

빠른 기간 선택(7일·1개월·3개월) & 자동 날짜 계산 기능

상세 기간 선택 기능 + DatePickerModal 연동: 선택된 날짜는 그대로 SearchPeriod 내부 상태에 반영되며, start / end 변경을 부모 컴포넌트로 전달하는 구조(onChange) 덕분에 재사용성이 높음.

TimePicker 컴포넌트

모바일 친화적인 시간 선택 UI 구현 (ScrollSelect 기반): TimePicker는 시간과 분(minute)을 스크롤 방식으로 선택할 수 있도록 구성하여 모바일 환경에서도 직관적이고 사용하기 쉬운 네이티브 타임피커 같은 UX를 제공한다.
시간/분 리스트는 ScrollSelect 컴포넌트로 분리하여 관리하므로, 선택값이 자연스럽게 중앙에 고정되는 스크롤 인터랙션을 구현할 수 있었다.

(1) 메인 페이지 및 전역 레이아웃(헤더·푸터·사이드바) 개발

메인 API를 이용한 레이아웃 설계

  • 메인 API에서 제공되는 헤더 로고, 알림 수, GNB 메뉴, 메인 위젯 데이터를 이용해 전체 레이아웃을 데이터 기반으로 설계.
  • 사용자 권한 값(allowedFormIds, ENV_SEQ_WIDGETS, groupEnabled)에 따라 위젯 노출/비노출 규칙을 동적으로 구성, 잘못된 formId 매핑은 useMemo 최적화를 통해 렌더 성능 개선.
  • 메인 API가 모든 데이터를 한 번에 제공하는 구조로 인해 페이지 단위 분리가 미흡하다고 판단하여, 설정 페이지 개발자와 협업해 설정 API와 메인 API를 결합하는 구조를 재정의.
  • 설정 API에서 권한 및 위젯 설정(on/off)을 먼저 검사하고 이후 메인 API 데이터를 바인딩하여 정확한 권한 기반 메인 페이지 구성을 구현
  • 설정 API의 바로가기(Shortcut) 설정을 메인 페이지에 반영

Header 설계

  • 메인 API의 로고 데이터를 동적으로 바인딩하여 기업별 커스터마이징 대응
  • 알림 API와 연동해 읽음/안읽음 상태를 UI에 반영
  • GNB 메뉴명을 언어팩에 통합 적용하여 공통 번역 관리가 가능하도록 구조화

Sidebar 설계

  • 사용자 사번 확인용 UPC 형태의 바코드 모달 기능 구현
  • 메인 API의 GNB 데이터를 기반으로 권한별 사이드바 메뉴 목록을 구성

Appbar 설계

  • 설정 API의 Appbar 데이터를 바인딩하여 사용자 맞춤형 앱바 구성 구현
  • 설정 데이터를 localStorage에 저장하고, 값 변경을 감지해 즉시 UI 업데이트가 이루어지도록 설계

(2) 테마, 언어팩 개발

ThemeProvider

  • ThemeProvider를 구성하여 전역 테마 상태를 중앙에서 관리하도록 설계
  • 테마 변경 값을 HTML 루트의 data-theme 속성에 반영하여 CSS 변수 기반 테마 전환 구현
  • CSS Custom Properties를 활용해 컴포넌트 수정 없이 전체 UI 컬러 일괄 변경 가능하도록 구조화
  • useTheme 훅을 제공해 하위 컴포넌트에서 간편하게 테마 접근 및 제어 가능하도록 설계

LanguageProvider

  • 기존 언어팩과 메뉴명이 백엔드에서 변경될 수 있는 구조를 고려해, 초기 진입 시 서버 응답 기준으로 언어팩 워딩을 동적으로 보정하여 UI 불일치 및 재렌더링을 최소화
  • LanguageProvider를 구성해 전역 언어 상태 및 다국어 메시지 관리를 중앙화
  • 초기 로딩 시 localStorage와 브라우저 언어를 기준으로 기본 언어 자동 감지 로직 구현
  • 메인 페이지에서 설정 API 응답을 기반으로 서버 설정 우선 언어 적용 처리
  • 기본 다국어 메시지와 동적 메뉴 데이터를 병합해 런타임 다국어 메뉴 확장 가능하도록 설계

(3) 메뉴 [급여조회] 개발

급여조회

  • 급여 라운드(Round)로 지급 회차 단위 급여명세 조회 기능 구현
  • 진입 시 최신 라운드를 자동 조회하여 기본 급여 기간 정보를 즉시 표시
  • ScrollSelect 컴포넌트 기반 라운드 선택 UI를 제공해 과거 급여 회차 조회 지원
  • 대상자 여부 및 급여 마감 여부 체크를 통해 조회 가능 대상자만 급여 노출 처리
  • 급여 총액·공제 총액·실지급액을 요약 테이블로 제공해 급여 핵심 정보 강조
  • RSA 공개키 기반 보안 파라미터를 적용해 급여명세서 출력 보안 강화
  • 귀속월(YYYY.MM) 기준 급여명세서 조회 페이지를 구성해 월 단위 급여 확인 기능 제공
  • 사용자 정보 기반으로 조회 파라미터를 자동 세팅해 개인별 급여 데이터 조회 처리
  • 급여 마스터 데이터를 통해 총지급액·공제액·과세/비과세 금액 요약 제공

월급여대장

  • 사용자별 급여 내역 조회 UI 구현
  • 기간 선택 컴포넌트를 통해 급여 귀속 월 범위 검색 기능 제공
  • 마감 월 체크 API를 연동하여 마감 이후 월 조회 및 출력 제한 로직 구현
  • 급여 항목을 다중 행 테이블로 구성하여 월별 급여 상세 내역 가독성 강화
  • 급여대장 출력 요청 시 보안이 적용된 리포트 출력 구현

(4) 메뉴 [연말정산] 개발

연말정산 검토사항

  • 연말정산 귀속년도 공통코드를 조회해 회사별 연말정산 기준 연도 자동 결정 처리
  • 연말정산 상태 조회를 통해 해당 연도의 연말정산 진행 가능 여부를 사전 판단
  • 연말정산 미대상 상태인 경우 검토 화면을 차단해 잘못된 접근 방지
  • 검토사항 목록을 조회하여 EY 검토 항목별 상세 내역을 카드 형태로 제공
  • 미확인 항목이 있을 경우 해당 항목을 자동으로 펼쳐 사용자 확인 흐름 유도
  • 검토 항목별 공제 항목·필요 서류·검토 의견을 명확히 분리해 정보 전달력 강화
  • 검토 결과 데이터를 데이터셋 형식으로 변환해 대량 저장 API와 연동

연말정산 증빙서류

  • 증빙서류 유형 단위로 SupportingFileItem 컴포넌트를 구성해 증빙 항목별 파일 관리 구조 구현
  • 연말정산 상태 및 기간 정보에 따라 업로드·삭제 가능 여부를 컴포넌트 단에서 제어
  • 파일 업로드 시 RSA 암호화 및 보안 헤더를 적용해 증빙서류 전송 보안 강화
  • 증빙서류 목록을 표시하고 파일 다운로드 기능을 제공해 제출 내역 확인 지원
  • 파일 삭제 시 확인 모달을 거쳐 처리해 실수로 인한 데이터 삭제 방지
  • 증빙 파일 미리보기 컴포넌트(ImagePreviewModal)을 제공해 업로드 파일 즉시 확인 가능
  • 증빙서류 설명 및 관련 문서 안내를 모달로 제공해 제출 기준 이해도 향상

연말정산 결과조회

  • 연말정산 결과 데이터를 조회해 환급/추징 여부 및 금액 요약 표시
  • 환급·추징 상태에 따라 아이콘·색상·문구를 분기 처리하여 가독성 강화
  • 연말정산 비교 출력 기능을 제공해 1·2차 결과 비교 PDF 출력 지원
  • 원천세율 설정이 가능한 경우 원천징수 세율 선택 및 저장 기능 제공
  • 분납 개월 수 및 원천세율 변경 시 저장 API 호출로 사용자 설정 반영
  • 출력 가능한 연말정산 서식 목록을 조회해 서식별 PDF 출력 지원

(4) 메뉴 [HELP DESK] 개발

헬프데스크 리스트

  • 게시판 설정 및 문의 유형 공통코드를 조회해 Help Desk 카테고리 필터를 동적으로 구성
