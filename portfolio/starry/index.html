(1) 스타일 시스템 — Tailwind CSS v4 & 다크/라이트 테마

v 4.0

  • 다크/라이트 테마 전환 구현 전략

  • CSS Custom Properties 기반 테마 설계
    Tailwind CSS v4의 dark: 유틸리티와 CSS 변수를 결합해, HTML 루트의 클래스 토글 한 번으로 전체 컬러 시스템이 전환되는 구조를 설계했습니다. 각 색상을 변수로 추상화하여 테마 추가 시 컴포넌트 코드를 건드리지 않아도 되는 확장 가능한 아키텍처를 구현했습니다.
  • JIT 기반 경량 빌드
    Just-in-Time 컴파일 방식으로 실제 사용하는 클래스만 번들에 포함시켜 CSS 용량을 최소화했으며, 유틸리티 클래스 조합으로 별도의 CSS 파일 없이 일관된 디자인 시스템을 유지했습니다.

(2) 퍼널 기반 회원가입 UX 설계

Toss 앱의 UX 패턴을 참고해, 입력 자리수가 고정된 정보(생년월일, 인증코드 등)의 경우 별도의 CTA 버튼 없이 입력 완료 즉시 다음 스텝으로 자동 전환되도록 퍼널(Funnel) 구조로 설계했습니다.

각 스텝을 독립적인 뷰 상태로 분리하여 히스토리 관리와 뒤로 가기 동작을 일관되게 처리했으며, 입력 필드 진입 시 적합한 키보드 타입(numeric / email)을 자동 호출하고 기본 Focus 상태를 설정해 사용자가 추가 조작 없이 입력에 바로 집중할 수 있도록 구현했습니다.

(3) 일기 유형 분기 & 뷰 모드 설계

질문형 / 기록형 두 가지 일기 타입을 공통 데이터 스키마 위에서 유형 필드(type)로 분기 처리해, 렌더링 컴포넌트만 교체하는 방식으로 유지보수가 용이한 구조를 설계했습니다.

각 일기에 감정(emotion) 값을 함께 저장하여 별 색상 분기의 데이터 근거로 활용했으며, 캘린더 뷰와 리스트 뷰는 동일한 데이터 소스를 쿼리 파라미터로 뷰 모드만 전환하는 방식으로 구현해 불필요한 API 재요청 없이 상태를 유지하도록 처리했습니다.

(4) PIN 기반 앱 잠금 구현

앱 진입 시 PIN 번호를 검증하는 잠금 레이어를 Next.js 미들웨어 수준에서 처리해, 인증되지 않은 상태에서는 어떤 라우트로 직접 접근해도 잠금 화면으로 리다이렉트되도록 구현했습니다.

PIN은 평문이 아닌 해시 처리 후 Supabase DB에 저장하여 보안을 강화했으며, 설정 화면에서 잠금 기능 ON/OFF와 PIN 변경을 별도 플로우로 분리해 사용자 실수로 인한 잠금 해제 혼란을 방지했습니다.

(5) Supabase 연동 아키텍처

Supabase Auth — 이메일/비밀번호 방식으로 회원가입·로그인을 처리하고, Next.js App Router의 Server Component에서 세션을 검증해 인증 상태를 서버 단에서 선제적으로 제어합니다.

Supabase DB (PostgreSQL + RLS) — 일기, 감정, 별 데이터를 테이블로 설계하고 Row Level Security 정책을 적용해 인증된 사용자만 자신의 데이터에 접근할 수 있도록 DB 수준에서 권한을 격리했습니다.

Server / Client Component 분리 — 데이터 패칭이 필요한 페이지는 Next.js Server Component에서 Supabase를 직접 조회해 초기 HTML에 데이터를 포함시키고, 인터랙션이 필요한 UI만 Client Component로 분리하여 불필요한 클라이언트 사이드 요청을 최소화했습니다.
